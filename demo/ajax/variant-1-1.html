The lightbox and fallbacks

Please keep in mind users use the web very differently!
It might be a user wants to open a lightbox-link in a new tab using e.g. the right click/context menu. In this case the lightbox will do ... nothing. It simply can't. So you have to care about a fallback!

In general you should build a server-side switch:
If any JavaScript does request content there will be an additional header field "X-Requested-With: XMLHttpRequest". So does the lightbox. If this field and value was set, you can respond with partial HTML. If not you should return a full page.

If you do not want to have a full page available (because you don't care about defaults and an easy to use web ;) consider to NOT use links for binding the lightbox. Use one of the multiple other possibilities to bind the lightbox!
